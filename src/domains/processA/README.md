# Process A — Outcome-driven Analysis

## Purpose
Observe pattern-based signals from job application outcomes.

## Key Assumption
Each application is submitted after JD-aware content review.
(Document quality is treated as controlled and excluded from analysis.)

## Input
CSV / Excel job application history (n ≥ 10 recommended)

## Output Principles
- Use likely / estimated language only
- Separate observation from interpretation

## Status Normalization
- NoResponse
- Rejected
- Interview
- Offer
- Withdrawn
